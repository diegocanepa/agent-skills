# MR/PR Templates

## Standard Template
```markdown
## Description
[Brief overview of what this change does and why it's needed]

## Related Issue
Closes #[issue-number]

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Refactoring
- [ ] Documentation update
- [ ] Other: [Description]

## Changes Made
- [Specific change 1]
- [Specific change 2]

## Testing
- [ ] Unit tests added/updated
- [ ] Integration tests added/updated
- [ ] Manual verification completed

## Screenshots (if applicable)
[Add screenshots for UI changes]
```

## Feature Template
```markdown
## Summary
Introducing [Feature Name] to allow [capability].

## Related Issue
Closes #[issue-number]

## Technical Details
- [Design decision 1]
- [Pattern/Library used]

## Verification
1. [Step 1 to verify]
2. [Expected result]
```

## Bug Fix Template
```markdown
## Description
Fixes [Problem] where [Actual Behavior] happened instead of [Expected Behavior].

## Related Issue
Fixes #[issue-number]

## Root Cause
[Brief explanation of why it was failing]

## Verification
- [ ] Reproduced before fix
- [ ] Verified after fix
```
